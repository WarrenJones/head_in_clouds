# Design Detail: Head in the Clouds / 云上心事

> 这是比 `DESIGN_BRIEF.md` 更硬的设计规格，用于避免 AI 设计工具自由发挥。  
> 如果只能上传一份文档给 Claude Design / Figma Make / v0，优先上传这一份。

## 1. 不可违背的产品约束

1. 用户第一动作是拍/传票证，不是输入航班号。
2. Post 不是纯文字，必须有系统生成 Cloud Card。
3. 票证图片不是 Post 内容，不能公开展示。
4. Stage 1 不支持用户视频。
5. 自由评论只在本航班开放。
6. 同旅程、目的地、热点不显示评论按钮、评论数、评论输入框。
7. 每个屏幕都必须移动端优先。
8. 不要做地图、航班状态、延误提醒、登机口信息。

## 2. Post UI 结构

### Feed Card

每条 feed card 从上到下：

1. `Cloud Card` 缩略图，占卡片视觉面积 45%-60%。
2. 航线 metadata：`MU5405 · 上海 SHA → 成都 CTU`
3. 心境标签：如 `奔赴`
4. 正文摘要：最多 2-3 行。
5. 随行照片提示：如果有照片，显示 `2 photos` 或小缩略图组。
6. 互动栏：
   - 本航班：点赞、收藏、评论、举报。
   - 非本航班：点赞、收藏、预设回应、举报。

### Post Detail

1. Cloud Card 大图。
2. 完整正文。
3. 航班 metadata。
4. 随行照片 0-3 张。
5. 互动区。
6. 评论区条件展示：
   - Same Flight：显示评论列表和 composer。
   - Other spaces：显示 `评论只在本航班开放`。

## 3. 评论组件规格

### Comment Composer

仅在本航班 Post Detail 出现。

元素：

- Placeholder：`给这趟飞行里的人留一句话`
- 字数提示：`0/140`
- 发送按钮：`发送`
- 提醒：`只有本航班乘客可评论`

不支持：

- 图片评论
- 视频评论
- 链接
- @用户
- 楼中楼

### Comment Item

元素：

- 匿名身份：`14F · 靠窗`
- 时间：`20 分钟前`
- 文本：1-140 字
- 操作：点赞、举报、删除自己的评论

## 4. 12 屏详细布局

### 01 Opening

布局：

- 顶部留白 56px。
- 中上：英文品牌，电影片名感。
- 中部：一张大 Cloud Card，略微倾斜。
- 下方：一句中文主文案。
- 底部：主 CTA + 次 CTA + 隐私提示。

核心文案：

- `拍下这趟飞行，留下只属于云上的心事`
- `票证原图默认不保存`

### 02 Capture

布局：

- 全屏暗色相机背景。
- 中部取景框，四角框线。
- 顶部关闭按钮。
- 底部拍摄按钮、相册入口。
- 辅助 chips：`登机牌`、`电子机票`、`航班截图`。

注意：

- 不出现航班号输入框。
- 备用文案只能是：`识别失败后可修正`。

### 03 Privacy Redaction

布局：

- 上方：上传图预览。
- 图上用半透明遮罩标出敏感区域。
- 下方：解释卡片。
- 底部 CTA。

文案：

- `先保护你的票证`
- `姓名、票号、证件号不会出现在任何 Post 里`
- `原图默认删除，只保留航班信息`

### 04 Unlocking

布局：

- 全屏动效感。
- 中间航线弧线。
- 逐步识别字段以 checklist 浮现。

字段：

- `MU5405`
- `2026-05-15`
- `上海 SHA`
- `成都 CTU`

### 05 Flight Confirm

布局：

- 顶部航线大标题。
- 中部 ticket-like 信息卡。
- 下方隐私说明。
- 底部 CTA。

CTA：`确认，这是我的这趟飞行`

### 06 Compose Post

布局：

- 顶部航班小条：`MU5405 · SHA → CTU`
- 大标题：`此刻，你想把什么留在云上？`
- 心境横向选择。
- Prompt chips。
- 正文输入区。
- 随行照片入口，最多 3 张。
- 底部 CTA。

输入区默认样例：

`我在起飞前最后一次看见这座城市的灯……`

### 07 Card Studio

布局：

- 上方大 Cloud Card。
- 中部模板 carousel。
- 下方操作区。

模板：

- 舷窗：云层 + 航线。
- 胶片：电影感边框。
- 邮戳：票根/邮戳感。
- 极简：大量留白。

### 08 Publish / Share

布局：

- 成功状态。
- Cloud Card 小预览。
- 分享渠道按钮。
- CTA：`进入本航班`

### 09 Flight Space

布局：

- 航班 header。
- 当前用户 Post 置顶。
- Feed 列表。
- 底部浮动按钮：`再写一张`

Feed card 必须有评论入口。

### 10 Discovery Spaces

布局：

- 顶部 tab。
- 当前空间说明。
- Feed cards。
- 锁定评论提示。

Feed card 不得有评论入口。

### 11A Post Detail From Same Flight

显示：

- Cloud Card 大图。
- 完整正文。
- 随行照片。
- 评论列表。
- 评论 composer。

### 11B Post Detail From Discovery

显示：

- Cloud Card 大图。
- 完整正文。
- 随行照片。
- 点赞、收藏、预设回应。
- `评论只在本航班开放`

不显示：

- 评论列表。
- 评论 composer。

### 12 Report / Block

布局：

- Bottom sheet 或 full screen modal。
- 举报原因列表。
- 提交按钮。
- 屏蔽选项。

## 5. 高风险设计误区

- 把它做成航班查询工具。
- 把它做成普通日记。
- 把 Post 做成纯文字。
- 在非本航班页面显示评论。
- 让票证上传看起来像 KYC。
- 用太多图标和状态字段破坏情绪。
- 用 AI SaaS 的紫色渐变。

## 6. 推荐生成顺序

1. 先生成 01-08：capture-to-card flow。
2. 单独生成 Cloud Card 组件 4 个模板。
3. 再生成 09-12：feed、post detail、comment、report。
4. 最后合并成 prototype。

不要一次性让 AI 生成完整 12 屏，否则它会压缩细节并自由发挥权限规则。
