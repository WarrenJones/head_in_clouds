## 0. 产品一句话

iOS micro App。用户先写一句话并生成私人 Cloud Card；航班信息可以稍后补。只有发布到同班机、评论和接收同班机新笔记通知时，才需要验证航班。

**核心体验逻辑**：飞行前锚定 → 表达先行 → 私人卡完成 → 验证后置 → 同班机发布 / 卡片扩散 / 同班机召回。

## 1. 不可违背的设计约束

1. iPhone mobile only，尺寸 390 × 844。
2. 背景为 dawn / cabin-light 氛围：深蓝灰、云白、舷窗金、纸张米白；不要紫色 AI SaaS 风。
3. 中文 serif + 英文 italic display + 航班信息等宽字体。
4. Cloud Card 是视觉中心，必须像诗歌海报/实体卡片，不像 App 截图。
5. 默认 Cloud Card 必须是复古明信片 / 票根信纸方向：深蓝底 + 米黄纸张 + 坐标路线 + 大字 quote。
6. 票证是 proof，不是内容主视觉；登机牌原图不公开。
7. 06 Compose 一句话即可完成，不要 200 字门槛。
8. 离线状态必须设计成核心状态，不是 error 辅助态。
9. 16 Shared Card Landing 是同班机笔记落地页，不是下载页。
10. Stage 1 不接 flight tracking API，不展示登机口、延误、实时航班状态。
11. Cloud Card 不出现“立即下载”、App Store badge 或强营销 CTA。
12. 不要全局深色 UI：深色只做氛围层，核心操作/阅读区用米白纸张或浅色卡片。
13. 微信绿 `#07C160` 只能出现在 14 微信登录按钮，不能用于 13 账号状态卡或主品牌色。
14. 真实 UGC 可能很普通，默认卡片必须能承接“飞机晚点了，累死”这类文本。
15. 不要把扫描登机牌放成首次写作前的硬门槛；扫描/验证只用于发布同班机。
16. 公开区域禁止具体座位号；用“靠窗的人 / 通道旁的人 / 同机乘客”等模糊身份。

## 2. 状态机

```text
打开 App
├── 未 onboarded → 00a → 00b → 00c → 00d → 01a
├── onboarded 但还没发布 → 01a
├── onboarded 且 ≥1 发布 → 01b
├── 从登机提醒进入 → 06 Compose（带 FlightIntent）
├── 从同班机新笔记通知进入 → 09 Flight Space（定位到新笔记）
├── 有离线未同步稿 → 06 / 08 同步状态，内容已本地保存
├── 有普通 draft → 06 Compose（继续上次）
└── Guest 第 2/3 张 Post 发完 → 09 上叠 08b → 14 登录升级
```

## 3. 23 屏清单

### 00a Welcome

- Dawn 氛围可以保留，但不要纯全屏深色；中间放米白明信片/票根式视觉锚点。
- 大字双语品牌：`Head in the Clouds` + `云·上·心·事`。
- 主 hook：`登机前 30 分钟，给这趟飞行留一句话`。
- 副 hook：`落地后，生成一张只属于云上的明信片`。
- 主 CTA：`写下这一趟`。
- 次入口：`添加航班信息`。
- 不要注册框、不要 feed。
- 目标：首次打开就让用户产生“原来飞行体验可以这样被记录”的顿悟。

### 00b Permission Primer

- 标题：`为了你的飞行心事，我们需要：`
- 三项权限：相机、相册、通知。
- 相机说明：`拍登机牌时申请`。
- 相册说明：`保存 Cloud Card 时申请`。
- 通知说明：`添加登机提醒时申请`。
- 主 CTA：`继续，不现在授权`。
- 说明：`以上权限均在你实际用到时才申请，不提前索权`。

### 00c Privacy Promise

- 标题：`你的票，只在你手里`。
- 三条：原图不存云、卡片不显示敏感字段、评论只在本航班可见。
- CTA：`我知道了，继续`。

### 00d Get Started

- 标题：`准备好了吗？`
- 副标题：`不需要注册，直接开始你的第一次`。
- 主 CTA：`开始我的第一次飞行`。
- 底部小字 link：`已有账号？登录` → 14。
- 底部细字：`之后可以选择绑定微信或手机号，换设备也能找回`。

### 01a Opening · First-published

- 大 Cloud Card sample，表现 route + quote + metadata。
- CTA 1：`写下这一趟` → 06。
- CTA 2：`添加航班号 / 扫登机牌` → 02。
- CTA 3：`添加登机提醒` → 15。
- CTA 4：`看看别人留下的` → 10。
- 轻提示：`航班信息可以稍后补。验证后才能进入同班机。`
- 底部小字：`票证原图默认不保存`。

### 01b Opening · Returning

- 右上 settings icon → 13。
- 顶部：`最近的云上心事` + 用户最新 Cloud Card 缩略。
- 中部：`准备好下一次飞行了吗？`
- CTA 1：`写下这一趟` → 06。
- CTA 2：`添加航班号 / 扫登机牌` → 02。
- CTA 3：`添加登机提醒` → 15。
- CTA 4：`我的飞行册 · {N}` → 09b。

### 02 Add Flight Info

- 手动航班号 + 日期是基础输入；登机牌 / 电子机票 / 行程截图是快捷验证方式。
- 暗色相机取景框，corner brackets。
- 文案：`对准登机牌、电子机票或航班截图`。
- 上传相册入口。
- 左上返回。
- 标题：`添加航班信息`。
- 说明：`写作不需要验证；发布到同班机时需要确认这趟航班。`

### 03 Privacy Redaction

- 票证预览 + 敏感区域示意。
- CTA：`我已遮挡` / `帮我打码`。
- 文案：`我们只提取航班信息，原图默认删除`。

### 04 Unlocking

- 云层 / 航线 / 坐标解锁动效感觉。
- 文案：`正在解锁这趟云上的坐标`。
- 逐步出现航班号、日期、出发地、目的地。

### 05 Flight Confirm

- 航班号、日期、航线；不要显示具体座位号。
- 信息可编辑。
- 未来航班或手动录入时，展示开关：`设置登机前 30 分钟提醒`，默认开启。
- CTA：`确认并发布到同班机` / `解锁同班机`，根据来源变化。

### 06 Compose · One-line first

- 标题：`这次飞行，我只想说：`
- 默认输入 placeholder：`比如：我把没有说出口的话，带过了云层。`
- 顶部航班状态 chip：
  - `未添加航班 · 稍后补`
  - `MU5301 · SHA → CTU`
  - `MU5301 · 已验证同班机`
- 允许一句话完成，不显示字数不足。
- 模板卡：
  - `飞往 ___，因为 ___，起飞时我 ___`
  - `如果能带一样东西上飞机，是 ___`
  - `落地以后，我希望 ___`
- 语音入口：按住说话 → 转文字 → 用户确认。
- 顶部状态三种：`草稿已保存` / `已离线保存，落地后自动同步` / `同步失败，内容已保存 · 重试`。
- 主 CTA：`生成私人明信片`。
- 次入口：`添加航班，解锁同班机`。

### 07 Card Studio · Cloud Card

- 大 Cloud Card preview，卡片比 App UI 更像艺术品。
- 默认模板必须采用复古明信片 / 票根信纸方向，不要默认用照片叠字。
- 必须显示：
  - 路线线条 / 坐标感。
  - 大字 quote。
  - 航班号、日期、心境小字。
  - 左下角极小 app icon / watermark。
  - 低调可识别入口，指向 16。
- 模板选择：`boarding_postcard`（默认）/ `route_poem` / `cloud_window`。
- 增加普通文本兜底模板：`flight_log` / `飞行日志`。它是深色票根或简洁便签，适合“飞机晚点了，累死。”，不要强行诗意化。
- `cloud_window` 只作为条件推荐模板，不要当默认；它依赖用户文案和照片足够诗意。
- 画一个普通 UGC 兜底状态：文本为 `飞机晚点了，累死。`，卡片仍然像克制的 flight note，不要审美坍塌。
- 操作：编辑 quote、换模板、保存预览、继续发布。
- 未验证状态也能生成卡片；metadata 显示 `航班待确认` 或用户手填路线。

### 08 Publish / Share

- 拆两个状态：
  - `Private Share`：不要求验证，只保存/分享私人卡。
  - `Same Flight Publish`：必须完成航班验证后发布到同班机。
- 显示在线 / 离线 queued / 同步失败状态。
- 主 CTA：`保存 / 分享私人卡`。
- 次 CTA：`验证航班并发布到同班机`。
- 提示：`私人卡不会进入同班机。`
- 分享入口：朋友圈、微信聊天、小红书、复制链接。
- 分享图不含下载 CTA，链接进入 16。
- 发布完成后进 09。

## 4. Cloud Card 视觉规范

### 默认方向

默认使用复古明信片 / 票根信纸：
- 深蓝外底。
- 米黄纸张主体。
- 顶部航班号、路线、日期、心境。
- 邮戳或低调品牌 stamp。
- 中部大字 quote。
- 下方坐标路线。
- 底部极小品牌和隐藏识别入口。

### 必须做

- 竖版分享图优先，比例 3:4 或 9:16。
- 背景可以是云层颗粒、纸张、舷窗光、路线线条。
- Quote 是最大视觉元素。
- Route 是第二视觉元素。
- 品牌极小，像唱片封底署名。
- 二维码 / 可识别入口必须低调，不破坏审美。
- 普通文本也要成立，例如 `飞机晚点了，累死。`

### 禁止做

- 手机 mockup。
- App UI 截图。
- “立即下载”。
- 大二维码。
- 默认照片叠字。
- 全屏金色云海作为默认模板。
- 登机牌真实姓名 / 座位 / 票号。
- 航班 dashboard。

### 样例内容

```text
Quote:
我把没有说出口的话，带过了云层。

Metadata:
MU5301 · SHA → CTU · 2026.05.19 · 奔赴

Route:
SHA 31.2304°N → CTU 30.5728°N
```
