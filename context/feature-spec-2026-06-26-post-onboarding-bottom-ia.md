# Feature Spec: Post-onboarding Bottom IA
**日期**：2026-06-26  
**优先级**：P0  
**预计影响**：修复 01a/01b 首页信息架构偏离，提升 `compose_started / landing_viewed`，降低回访疲劳。

## 问题（用户视角）
用户完成 onboarding 后看到的是一张重复的大样例卡和一组同权重大按钮，像功能菜单，不像一个每天可打开的私人飞行心事 App。

## 根因（产品视角）
实现把 PRD 里的 01a/01b Opening 当作一次性 landing page，而不是 post-onboarding app shell。添加航班、登机提醒、飞行册、发现被做成同级主按钮，稀释了“写下这一趟”的核心动作，也让回访用户每次看到同一张样例图。

## 方案（What，不是 How）
post-onboarding 使用 3 个长期目的地 + 1 个中心创作动作的底部 IA：

- `今天`：默认首页。展示当前/最近状态、最近一张心事或空状态、轻量上下文动作。
- `写下`：中心主动作，直接进入 06 Compose，全 App 最高权重。
- `飞行册`：我的历史心事卡和航班记录。
- `发现`：别人留下的、同旅程/目的地/热点/随机。
- `设置`：右上角入口，不进入底部导航。

`添加航班号 / 扫登机牌` 和 `添加登机提醒` 只作为 `今天` 状态卡里的上下文动作出现，不再是首页常驻大按钮，也不进入底部 tab。

## 不做什么（防 scope 蔓延）
- 不新增第 5 个底部 tab。
- 不把设置放进底部 tab。
- 不重做 06/07/08 的卡片视觉。
- 不引入实时航班追踪、登机口、延误查询。
- 不为了填充发现页制造假内容。

## 验收标准（Acceptance Criteria）
- [ ] AC1：完成 onboarding 后进入 post-onboarding shell，底部可见 `今天 / 写下 / 飞行册 / 发现`，没有 5 个 tab。
- [ ] AC2：`写下` 是中心最高权重动作，点击后直接进入 06 Compose，不要求航班号、登机牌或权限。
- [ ] AC3：首页不再展示同一张大样例 Cloud Card 作为每次回访主视觉；有内容时展示最近心事的小摘要，无内容时展示轻量空状态。
- [ ] AC4：首页不再出现 `添加航班号 / 扫登机牌`、`添加登机提醒`、`我的飞行册`、`看看别人留下的` 四个同权重大按钮。
- [ ] AC5：`飞行册` 和 `发现` 可从底部入口到达，且不需要通过首页按钮。
- [ ] AC6：设置/账号入口仍在右上角，且可进入 13 Account & Settings。
- [ ] AC7：真机或模拟器截图确认 01a/01b 不是 landing-page 式功能菜单。

## 成功指标（上线后 2 周内衡量）
- 写作启动率：`compose_started / (01a_landing_viewed + 01b_landing_viewed)` 从当前 baseline 提升到 ≥ 45%。
- 回访写作启动：`compose_started(source=01b) / 01b_landing_viewed` 不低于首次态的 70%。
- 发现误触降低：`discovery_viewed / landing_viewed` 不作为首页按钮误触来源，只来自底部 `发现` 或上下文入口。
