## Prompt（复制下面整段）

```text
基于上传的 FIGMA_MAKE_BRIEF.md v2.4，重做 Head in the Clouds / 云上心事的核心体验屏：00a / 01a / 02-05 / 06 / 07 / 08 / 15 / 16。

这轮目标不是补屏数量，而是解决四个问题：
1. 首次打开还没有让我产生“原来飞行体验可以这样”的顿悟。
2. Cloud Card 需要从“好看的照片叠字”收敛为“任何真实 UGC 都能承接的默认分享资产”。
3. 主路径必须从“扫描登机牌优先”改成“先写一句，航班信息稍后补”。
4. 公开区域不能出现具体座位号，避免破坏隐私承诺。

必须优先遵守：
- 默认 Cloud Card 用复古明信片 / 票根信纸方向：深蓝底 + 米黄纸张 + 航班号信息 + 大字句子 + 坐标路线 + 邮戳感。
- 不要默认照片叠字，不要默认金色云海。照片叠字只做可选模板。
- 一句话就是完成态，不要 200 字门槛。
- 写作和私人 Cloud Card 不需要验证航班；同班机发布、评论、通知才需要验证。
- 深色只做氛围层；核心操作和阅读区域用米白、纸张、浅色卡片承载。
- 微信绿 #07C160 只能出现在 14 微信登录按钮；本轮如果出现账号状态，不要用大面积微信绿。
- 公开身份不能使用 14A / 21C / 7F / 33B，改为“靠窗的人 / 通道旁的人 / 同机乘客”。

需要重做的屏幕：

00a Welcome：
- 目标：首次打开就击中用户，而不是只展示品牌。
- 主 hook 必须是：“登机前 30 分钟，给这趟飞行留一句话”
- 副 hook：“落地后，生成一张只属于云上的明信片”
- 视觉：深蓝 dawn 氛围背景 + 中间米白明信片/票根式视觉锚点。
- 主 CTA：“写下这一趟”
- 次入口：“添加航班信息”
- 不要注册框，不要 feed，不要航班 dashboard。

01a Opening · First-published：
- 给还没发过 Post 的用户。
- 展示一张默认复古明信片 Cloud Card sample，不是照片叠字。
- 主 CTA：“写下这一趟”
- 次 CTA：“添加航班号 / 扫登机牌”
- 次 CTA：“添加登机提醒”
- 次 CTA：“看看别人留下的”
- 轻提示：“航班信息可以稍后补。验证后才能进入同班机。”
- 底部小字：“票证原图默认不保存”

02-05 Add Flight Info / Verify Flight：
- 这组屏幕不要作为首次写作前置门槛，而是“解锁同班机”的后置流程。
- 02 标题：“添加航班信息”
- 手动航班号 + 日期是基础输入；登机牌/电子机票/航班截图是快捷验证。
- 03 保留隐私说明，但不要重复承诺后又展示座位号。
- 05 不显示具体座位号。
- CTA：“确认并发布到同班机”或“解锁同班机”。

06 Compose · One-line first：
- 标题：“这次飞行，我只想说：”
- 输入框 placeholder：“比如：我把没有说出口的话，带过了云层。”
- 展示三种入口：一句话、模板、语音转文字。
- 模板卡：
  “飞往 ___，因为 ___，起飞时我 ___”
  “如果能带一样东西上飞机，是 ___”
  “落地以后，我希望 ___”
- 状态：草稿已保存 / 已离线保存，落地后自动同步 / 同步失败，内容已保存 · 重试。
- 顶部航班状态 chip：
  “未添加航班 · 稍后补”
  “MU5301 · SHA → CTU”
  “MU5301 · 已验证同班机”
- 操作区请用米白纸张或浅色卡片，不要全黑输入界面。
- 主 CTA：“生成私人明信片”。
- 次入口：“添加航班，解锁同班机”。

07 Card Studio · Default Cloud Card：
- 重点屏。默认 Cloud Card 必须参考“复古明信片 / 票根信纸”方向。
- 默认卡片结构：
  顶部：MU5301 / SHA → CTU / 2026.05.19 / 奔赴
  右上：低调邮戳或品牌 stamp
  中部：大字 quote “我把没有说出口的话，带过了云层。”
  下部：坐标路线，SHA 31.2304°N → CTU 30.5728°N
  底部：极小 Head in the Clouds / 云上心事 + 低调识别入口
- 模板选择：boarding_postcard（默认）/ route_poem / cloud_window。
- cloud_window 只能作为可选模板，不要默认展示。
- 请同时画一个普通 UGC 兜底状态：quote 是“飞机晚点了，累死。”，卡片仍然应该成立，像一张克制的 flight note，而不是审美坍塌。
- 普通 UGC 兜底不要继续套诗意明信片。新增/改造一个 flight_log / 飞行日志模板：深色票根或简洁便签，适合“飞机晚点了，累死。”、“这趟只想睡觉。”、“落地还有会。”。
- 未验证状态也能生成卡片；metadata 显示“航班待确认”或用户手动输入的路线。

08 Publish / Share：
- 拆成两个状态：
  1. Private Share：不要求航班验证，只保存/分享私人卡。
  2. Same Flight Publish：必须完成航班验证后发布到同班机。
- 默认展示复古明信片 Cloud Card。
- 主 CTA：“保存 / 分享私人卡”
- 次 CTA：“验证航班并发布到同班机”
- 提示：“私人卡不会进入同班机。”
- 分享入口：朋友圈 / 微信聊天 / 小红书 / 复制链接。
- 画离线 queued 状态：内容已保存，落地后自动同步。
- 分享图内不要“立即下载”，不要 App Store badge。

15 Flight Reminder Setup：
- 标题：“把这趟飞行留住”
- 航班号输入 MU5301，日期默认今天，起降城市可选。
- 提醒时间默认“登机前 30 分钟”，可改 15/45 分钟。
- 通知预览：“你要登上 MU5301 了，今天有什么想留下来的？”
- CTA：“设置提醒”
- 次 CTA：“现在就写一句”
- 不要画登机口、延误、航班状态、地图追踪。
- 页面应像轻量提醒，不像航班管理工具。

16 Shared Card Landing：
- 首屏展示默认复古明信片 Cloud Card，不是 App 下载页。
- 下方：“这张卡来自 MU5301 · SHA → CTU”
- 展示同班机笔记 1-3 条。
- 不展示具体座位号。
- CTA 1：“看同班机笔记”
- CTA 2：“我也飞过 / 下次也要飞这条航线”
- 不要“立即下载”，不要 App Store badge，不要强营销下载页。

视觉要求：
- 深蓝、米黄、纸张、低饱和蓝灰、少量舷窗金。
- 中文 serif + 英文 italic display + 航班信息等宽字体。
- 不要紫色 AI SaaS 风。
- 不要全局黑底 UI。
- 不要把设计做成微信插件界面。

交付：
- 只输出本轮指定核心屏及必要状态变体，不要重画整套。
- 07 必须输出三个状态：诗意 quote / 普通 UGC “飞机晚点了，累死。” / 未验证航班状态。
- 08 必须输出三个状态：Private Share / Same Flight Publish / 离线 queued。
- 每屏顶部 label 用屏号 + 名称。
```

## 跑完后的检查清单

- 00a 是否在 3 秒内讲清“飞行前提醒 + 一句话 + 云上明信片”。
- 07 默认卡片是否明显接近复古明信片，而不是照片叠字。
- 普通 UGC “飞机晚点了，累死。”是否进入 flight_log 兜底模板而不尴尬。
- 06 是否让人敢开始写，而不是像长文编辑器。
- 06 是否允许未添加航班时直接写一句并生成私人明信片。
- 15 是否轻，不像航班工具。
- 16 是否是同班机笔记落地页，不是下载页。
- 09 / 10 / 11 / 16 是否完全没有具体座位号。
